Any dispatch object contains event handlers for the specific chart element. Check Live Edit page in extended mode and find all events in json tree that are available for the specific chart. General dispatch options contain only general chart events. For example, for line chart: 
chart: {
    ...
    // general chart and tooltip events
    dispatch: {
        stateChange: function(e){ console.log('stateChange') },
        changeState: function(e){ console.log('changeState') },
        tooltipShow: function(e){ console.log('tooltipShow') },
        tooltipHide: function(e){ console.log('tooltipHide') },
        renderEnd: function(e){ console.log('renderEnd') }
    }
    ...
}
If you want to get and handle events for a specific chart element, you can do this via basic dispatch options. For our line chart example, this can be done as follows: 
chart: {
    ...
    // line chart events
    lines: {
        dispatch: {
            elementClick: function(e){ console.log('click') },
            elementMouseover: function(e){ console.log('mouseover') },
            elementMouseout: function(e){ console.log('mouseout') },
            renderEnd: function(e){ console.log('renderEnd') }
        }
    },

    // legend events
    legend: {
        dispatch: {
            legendClick: function(e) {...},
            legendDblclick: function(e) {...},
            legendMouseover: function(e) {...},
            legendMouseout: function(e) {...},
            stateChange: function(e) {...}
        }
    }
    ...
}
Note that the chart events are different from the global events (that not actually relate to the chart, but directive), described in Attributes section.
